Import Seasons Pass Holders from an Excel Spreadsheet.

Seasons Pass holders must match pre-existing License Holders in the database.
Seasons Pass Holders are identified by their License, or alternatively, their LastName, FirstName and optionally Date of Birth.
If you have the License code, use that for identification as it is guaranteed unique. If you don't have the License Code, you can use the Last Name, First Name and optionally the Date of Birth (if you have it) to match the License Holder. Be aware that by not using the License code, some License Holders may not match (say, due to spelling errors) and/or there could be more than one match (if two people have exactly the same name).

When using LastName and FirstName, capitalization and accents are ignored in the search. Other than that, the name must match exactly. For example, Joseph != Joey != Joe, Bob != Robert, Sue != Susan, Kris != Chris, etc.

Check for import errors carefully. Failed matches, or multiple matches, and those riders will not be added to the Season's Pass list.

The spreadsheet must have a specific format:

  • The data must be the first Sheet of the spreadsheet
  • The first Row of the sheet must be a Header Row.
  • Columns in the Header Row can be in any order.
  • The Header Row must contain a License column if you are importing by License.
  • The Header Row can contain Last Name, First Name and ideally, Date of Birth if you don't have the License code.
  • Addition column headers are ignored.

The same Seasons Pass data sheet can be imported any number of times without problems. It will not cause duplicate entries.
